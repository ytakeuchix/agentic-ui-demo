# Agentic UI Demo - 経費精算AIエージェント

Agentic UI (AI主導型UI)を体験するためのデモアプリケーションです。

> **"Maker (作業者) から Manager (監督者) へ"**
>
> 従来のCRUD操作のような「人が主導し、ツールを使う」関係ではなく、AIが自律的にタスクを処理し、人間は必要な時だけ介入する「人がAIを監督する」新しいUXデザインを実装します。

## コアコンセプト (Core Concepts)
このデモは、人間の介入レベルに応じた3つのUIモードで構成されています。

### 1. ZeroUI (Proactive Interface)
**"AIがすでに処理を終えている状態"**
AIは日次の経費処理を完了させ、結果だけを報告します。
問題がなければ、ユーザーは「確認完了」のステータスを見るだけで済みます。人間がトリガーするのではなく、AIが能動的にユーザーに働きかけます。

### 2. GenUI (Generative Interface)
**"文脈に応じて生成される、使い捨てのUI"**
入力不備や確認事項がある場合、AIはその問題を解決するためだけの最小限のフォームを動的に生成します。
巨大な固定フォームは存在せず、チャットストリームの中で自然にタスクを消化できます。

### 3. Canvas (Deep Verification)
**"AIとの共創・検証ワークスペース"**
複雑な判断（例：海外出張の監査や高リスクな経費）が必要な場合、画面が拡張され「Canvas」モードになります。
左側に「AIの推論・計画」、右側に「根拠となる事実（領収書、ログ等）」を並べ、AIがなぜそう判断したのかを透明化し、人間が納得して承認できる環境を提供します。

## デモシナリオ (Demo Scenario)

アプリケーションを起動すると、以下のストーリー順に体験が進みます。

### Scene 1: Notification (ZeroUI)
チャットボットが「昨日の経費精算処理結果」を能動的に通知してきます。
- ✅ 正常な経費はすでに処理済み
- ⚠️ 確認が必要な経費だけがハイライト表示される

### Scene 2: Correction (GenUI)
単純な情報不足（例：同席者の入力漏れ）があるケース。
- 「確認・修正」ボタンを押すと、その場に**同席者入力専用フォーム**が生成されます。
- AIがカレンダー履歴から候補者を推論し、ワンタップで補完できます。

### Scene 3: Inference (GenUI)
AIが推論を行ったが、確信度が低いため人間に確認を求めているケース（例：Uberの休日利用）。
- フォーム入力ではなく、AIが提示した**「推論カード（休日出勤の可能性）」**に対して「はい/いいえ」で答えるだけのUIが生成されます。
- Slack等の外部データソースを根拠として引用提示します。

### Scene 4: Verification (Canvas)
複数の情報源を照合する必要がある複雑なケース（例：海外出張における不正リスクの監査）。
- GenUIでは扱い切れないため、**Canvasモード**へシームレスに遷移します。
- 地図、タイムライン、領収書、為替レートなどを統合表示し、AIの承認推奨理由を検証します。
