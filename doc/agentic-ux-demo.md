# Agentic UI Demo Overview

このデモは、次世代のAIエージェントとの協働体験（Agentic UX）を具現化したプロトタイプです。
ユーザーは従来の「入力作業者（Maker）」から、AIの成果物を確認・承認する「監督者（Manager）」へと役割がシフトします。

## コアコンセプト

| コンポーネント | 役割 | 従来UIとの違い |
| :--- | :--- | :--- |
| **1. ZeroUI**<br>(Proactive UI) | **報告**<br>ユーザーが頼む前に仕事を完了させ、結果だけを通知する。 | トリガーが「ユーザー」ではなく「AI」。<br>ログインして確認しに行く必要がない。 |
| **2. GenUI**<br>(Generative UI) | **修正**<br>問題解決に必要な最小限のUIをその場で生成する。 | 巨大な固定フォームが存在しない。<br>文脈に応じて「同席者入力」や「推論確認」など形が変わる。 |
| **3. Canvas**<br>(Explainable UI) | **検証**<br>AIの思考プロセスと根拠（エビデンス）を可視化する。 | 「承認画面」ではなく「推論のぞき穴」。<br>ソース（領収書、Slack、カレンダー）を横断して確認できる。 |

---

## デモシナリオ詳細

### 1. ZeroUI: 日次サマリー通知 (Notification)
チャットボットが能動的に「昨日の経費処理結果」を持ってきたシーンから始まります。

*   **特徴**:
    *   **Visual Provenance**: 各経費行に 🧾(Receipt) や 📅(Calendar) のアイコンを表示し、AIがどこから情報を拾ったかを可視化しています。
    *   **Instant Result**: 正常なものは「処理済み」として報告。エラーがあるものだけが強調表示されます。

### 2. GenUI: 動的修正フォーム (Correction)
エラー項目（要確認項目）の「確認・修正」ボタンを押すと、AIがそのケース専用のフォームを生成します。

*   **Data Handling Strategy**:
    *   **Hidden by Default**: 経費精算には通常20項目以上（税区分、部門コード、インボイス番号等）が必要ですが、AIが確信度高く補完できた項目は初期表示しません。ユーザーの認知負荷を「確認が必要な1点」に集中させます。
    *   **Full Disclosure Option**: 必要であれば「詳細を表示」ですべての補完項目と推論根拠を展開可能です。

*   **演出**: フォーム展開時に「Generating...」のアニメーションを表示し、動的生成感を強調しています。

#### ケースA: 問題なし（Salesforce研修受講）　　
*   **状況**: 教育費。Salesforce研修受講
*   **GenUIの解**: 事前の研修受講の申請が承認されているため、問題なしと判断。
*   **AIアシスト**: 特になし

#### ケースB: 情報不足（ステーキハウス銀座）
*   **状況**: 交際費だが、連携したGoogleカレンダーに予定がなく同席者が不明。
*   **GenUIの解**: 「同席者リスト」の入力フォームのみを生成。
*   **AIアシスト**: 過去の履歴やカレンダーから「候補者（内部・外部）」を提案し、ワンタップで入力完了させます。

#### ケースC: AI推論の確認（Uber交通）
*   **状況**: 休日の移動履歴。カレンダー予定はないが、Slackで緊急対応の会話があった。
*   **GenUIの解**: 入力フォームではなく「推論の提示（Inference Card）」を生成。
*   **AIアシスト**: Slackのログ（`#dev-alerts` "Server down..."）と、勤怠システムの休日出勤申請（申請件名とダミーURLを記載）を引用し、「これは緊急対応ではないですか？」と問いかけます。ユーザーは「はい」を押すだけで完了します。

### 3. Canvas: AI推論・エビデンス詳細確認 (Verification)
GenUIの提示だけでは判断がつかない、複数の情報源を照合する必要がある「認知負荷が高い」ケースで使用します。

#### ケースD: 海外出張の行動ログ突合（Complex Verification）
*   **状況**: サンフランシスコ出張に伴う、航空券・ホテル・現地移動（Uber）・食事が混在する大量の明細。
*   **認知負荷**: 「このUberは空港移動か？私用か？」「この食事は日当範囲内か？」などを1つずつカレンダーと照合するのは困難。
*   **AIアシスト**: GenUIでの対応は認知負荷が高いと判断し、ZeroUI通知の段階で関連明細をグルーピング（Trip Cluster）して提示し、そこからCanvasへ遷移させます。
*   **Canvasの解**:
    *   **Timeline View**: 出張全期間（Itinerary）と全決済（Action）を時系列リストで表示。
    *   **Geo-Matching**: 詳細な地図UIは使用せず、抽象的なルート可視化とホテル・オフィス周辺の「Zone Check」結果（In Zone / Out of Zone）を表示。
    *   **Cluster Verification**: 単一明細ではなく「出張イベント全体（フライト、ホテル、食事等）」をまとめて表示し、一括承認（Approve Entire Trip）が可能。
    *   **Full Data Verification**: 隠蔽されていた全項目（レート換算、現地税区分など）もこの画面ではフルオープンされ、ソース（領収書原本）と突き合わせて確認できます。

## 技術的ハイライト

1.  **Context-Aware**: エラーの種類（Violation vs Mismatch）によって、提示するUIコンポーネントをReactレベルで切り替えています。
2.  **Visual Cues for Trust**: 「なぜそう判断したか」を常にアイコンや引用で示すことで、ブラックボックス化を防ぎ、ユーザーの信頼（Trust）を獲得する設計です。
3.  **Seamless Transition**: チャット（ZeroUI）→ カード（GenUI）→ 全画面（Canvas）と、介入レベルに応じてシームレスに画面遷移します。
